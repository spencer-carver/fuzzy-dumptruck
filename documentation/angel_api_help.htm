API Functions

All API calls return an XML stream with top top-level element of <result/>. If an error occurs, the API will return an <error/> element inside the <result/> whose contents will contain the error string. If the function call is successful, the API will return a <success/> element inside of the <result/> element whose contents may contain additional information about the result of the request. Some functions will return additional sub-elements of the <result/> element with the requested data. These elements are described in the function definitions that follow. For all section file management functions, the paths to folders and files must always be specified as relative to the root folder for the specified section.

CREATE_ACCOUNT(AccountID, User, Password, FirstName, LastName, MiddleName, Prefix, Suffix, Nickname, EMail, Homepage, SystemRights, AccountGroup, AccountExpires, Tel1Type, Tel1, Adr1Type, Adr1Line1, Adr1Line2, Adr1Line3, Adr1POBox, Adr1City, Adr1State, Adr1PostalCode, Adr1Country, Tel2type, Tel2, Adr2Type, Adr2Line1, Adr2Line2, Adr2Line3, Adr2POBox, Adr1City, Adr2State, Adr2PostalCode, Adr2Country, Tel3Type, Tel3, Tel4Type, Tel4, Title, Role, Organization, Department, SSN, DefaultViewableBy, ProfileViewableBy, OrganizationViewableBy, EmailViewableBy, WWWViewableBy, Telephone1ViewableBy, Telephone2ViewableBy, Telephone3ViewableBy, Telephone4ViewableBy, Address1ViewableBy, Address2ViewableBy)
Creates a new account based on the provided information and returns success or error result information. 
UPDATE_ACCOUNT(User, Password, FirstName, LastName, MiddleName, Prefix, Suffix, Nickname, EMail, Homepage, SystemRights, AccountGroup, AccountExpires, Tel1Type, Tel1, Adr1Type, Adr1Line1, Adr1Line2, Adr1Line3, Adr1POBox, Adr1City, Adr1State, Adr1PostalCode, Adr1Country, Tel2type, Tel2, Adr2Type, Adr2Line1, Adr2Line2, Adr2Line3, Adr2POBox, Adr1City, Adr2State, Adr2PostalCode, Adr2Country, Tel3Type, Tel3, Tel4Type, Tel4, Title, Role, Organization, Department, SSN, DefaultViewableBy, ProfileViewableBy, OrganizationViewableBy, EmailViewableBy, WWWViewableBy, Telephone1ViewableBy, Telephone2ViewableBy, Telephone3ViewableBy, Telephone4ViewableBy, Address1ViewableBy, Address2ViewableBy)
Updates an account record based on the User value provided.  Returns success or error result information. 
CHECK_ACCOUNT(User)
Checks to see if an account exists for the provided username.  Returns true or false based on existence.
ENROLL_USER(User, Section, SectionRights)
Enrolls a user into a specific course with the provided information.  Returns success or error result information.
DROP_ENROLLED_USER(User, Section)
Drops a user from a specific course.  Returns success or error result information.
CHECK_USER_ENROLLMENT(User, Section)
Checks to see if a specific user is enrolled in a given course.  Returns true or false based on enrollment.
CREATE_UPDATE_SECTION(Course_ID, Course_Type, Category, Instructor_ID, Instructor, Title, Keywords, Semester, Campus, School, Department, Course, Section, Cost, Credits, Contact1, Contact2, Enrollment_Begins, Enrollment_Ends, Class_Begins, Class_Ends, Hidden, Disabled, Viewable_By, Billing_ID, Enrollment_PIN, Description)
Creates a new course based on the provided information and returns success or error result information.
CHECK_SECTION(Section)
Checks to see if a specific course exists.  Returns true or false based on existence. 
AUTHENTICATION_PASS(User, Section, Password, Validate)
Creates a login token for single sign-on authentication.  Returns the single sign-on URL with token or error result information.
QUIZ_QTI_IMPORT(Section, Parent, ServerBase, BasePath, Filename, Data)
Creates new quiz questions from a QTI data source. Use Filename to provide the path to a QTI data file, relative to the root section root folder, or provide the QTI data directly via the Data parameter. The ServerBase parameter should contain the appropriate URI to the root directory of the server. This path will be added to image URIs that begin with a "/". The BasePath parameter should contain the relative path to the directory where associated images are located. This path will be added to any image URIs that do not begin with "http" or a "/".
QUIZ_SETTINGS(Section, Entry)
Retrieves the Advanced Quiz Settings for the specified quiz. Results are returned in a <quiz_settings/> element. 
QUIZ_SETTINGS_SAVE(Section, Entry, Data)
Saves changes to the Advanced Quiz Settings for the specified quiz. Data should contain a valid <quiz_settings/> element with the settings to be changed unspecified settings will not be modified. To remove a previously specified setting specify the setting with an empty string value. 
SECTION_ENTER(Section)
Initializes the specified course, group or library for access. This is only required if you wish to interact directly with script pages in the /Sections/ directory of the ANGEL application.  
SECTION_EXIT()
Destroys session state information created by the SECTION_ENTER function call, effectively exiting the section. 
SECTION_ROSTER(Section)
Returns roster information for the specified course, group or library.
Return Value: A <roster/> element with a subordinate <rec/> element for each enrollment entry for the specified section.
SECTION_TEAM_ROSTER(Section, Team)
Returns team roster information for the specified team.
Return Value: A <roster/> element with a subordinate <rec/> element for each member of the specified team in the specified section.
SECTION_TEAMS(Section)
Returns a list of teams defined for the specified course, group or library.
Return Value: A <teams/> element with a subordinate <rec/> element for each team in the specified section. 
SECTION_SEARCH(Section, User, Find, Types, Filter, Order, Fields, MaxRecs)
Returns a list of items in the specified course, group or library that meet all specified criteria. 
Return Value: An <items/> element with a subordinate <rec/> element for each item that matches the criteria. 
SECTION_TOC(Section, User, Entry, Types, SkipNodes)
Returns a threaded listing of qualifying content items in the specified course. Because the results are threaded, items of type "FOLDER" will always be included in the results. If no Types are explicitly listed all top-level types are automatically returned. Sub-types such as "QUESTION" and "FIELD" items are never returned. To access these items use the SECTION_SEARCH function. If ENTRY is specified it must reference a valid Folder in the specified course, group or library. If specified, the returned results will contain only the child nodes of the specified folder. If not specified, all items within the section are returned.
Return Value: A <toc/> element with any number of <node/> elements which may themselves contain any of the following sub-elements: <title/> with the title of the node, <subtitle/> with the subtitle of the node and <children/> which may contain any number of node entries representing its descendant tree structure.
SECTION_USER_TEAMS(Section, User)
List the teams in which the specified user is a member in the specified course, group or library.
Return Value: A <teams/> element with a subordinate <rec/> element for each team in the specified section in which the specified user is enrolled. 
SECTION_IMS_IMPORT(Section, Parent, RelPath, Organization, FileType)
Imports the specified IMS file creating ANGEL items for all of the items specified in the imsmanifest file's organization. These items will be created as children of the item specified in the Parent parameter. If no Parent parameter is specified, the ROOT element is assumed. 
SECTION_ITEM(Section, User, Entry)
Retrieves metadata for the specified content item. The item must exist in the specified course.
Return Value: An <items/> element with a subordinate <item/> element with attributes for each metadata element. 
SECTION_ITEM_CREATE(Section, Data)
Creates a new content item in the specified course, group or library based on the settings specified. Settings should be passed in the Data element as a valid <item/> element with attributes for each of the settings you wish to specify.
SECTION_ITEM_SAVE(Section, Data)
Saves changes to the specified content item in the specified course, group or library. Which item is being updated is determined by looking at the item's ENTRY_ID attribute. The ENTRY_ID, OBJECT_ID and COURSE_ID settings cannot be changed. If the specified item is not in the course, group or library specified by the Section parameter an error occurs.
SECTION_FILES_CREATE_FILE(Section, Filename, Data, Unicode, Force)
Creates a new text file with its contents being the text passed in the Data parameter. If Unicode is 1 the file is saved as Unicode, otherwise as ASCII. The filename should be the path to the new file relative to the section root folder.
SECTION_FILES_CREATE_FOLDER(Section, RelPath)
Creates a new folder in the specified section. The RelPath parameter should contain the path for the new folder relative to the section root folder. 
SECTION_FILES_DELETE_FILE(Section, Filename, Force)
Deletes the file specified by Filename. The Filename parameter should contain the path for the file relative to the section root folder.
SECTION_FILES_DELETE_FOLDER(Section, RelPath, Force)
Deletes the folder specified by RelPath. The RelPath parameter should contain the path for the folder relative to the section root folder.
SECTION_FILES_EXTRACT_FILE(Section, Filename, DestPath, Password, Force, Delete)
Extracts the zip archive file specified by Filename to the folder specified by DestPath. Both of these settings should be relative to the section root folder. If the archive is password protected, provide the password using the Password parameter. If you wish to delete the archive after extraction set the Delete parameter to 1.
SECTION_FILES_LIST(Section, RelPath, Recursive)
Returns a list of folders and files in the folder specified by RelPath. RelPath should be relative to the section root folder. If the Recursive parameter is set to 1 information about all files and folders in subordinate folders is also returned. The root folder's contents can be returned by specifying "/" for the RelPath.
Return Value: A <folder/> element with subordinate <folder/> and <file/> elements for each folder or file contained in the directory. Subordinate <folder/> elements will contain the same if Recursive is set to 1. 
SECTION_FILES_MOVE_FILE(Section, Filename, DestPath, Force)
Moves the file specified by Filename to the new path and filename specified by DestPath. Both parameters should be relative to the section root folder. If DestPath points to a folder the file will be moved inside of that folder using the same filename.
SECTION_FILES_MOVE_FOLDER(Section, RelPath, DestPath, Force)
Moves the folder specified by RelPath to the new path specified by DestPath. Both parameters should be relative to the section root folder. If DestPath ends with a "/" or points to an existing folder, the folder will be moved inside of this folder. Otherwise, the folder will be created exactly as the specified DestPath. 
SECTION_FILES_RENAME_FILE(Section, Filename, NewName, Force)
Renames the file specified by Filename to the filename specified in NewName. NewName may not contain any path information. Filename should contain the path to the source file relative to the section root folder. 
SECTION_FILES_RENAME_FOLDER(Section, RelPath, NewName, Force)
Renames the folder specified by RelPath to the name specified by NewName. NewName may not contain any path information. RelPath should contain the path to the source folder relative to the section root folder. 
SECTION_FILES_FILES_UPLOAD_FILE(Section, RelPath, Filename, Force, UploadFile)
Uploads the file contained in the UploadFile element to the directory specified by RelPath. The Relpath should be relative to the section root folder. If Filename is specified the file will be saved as the specified name. If Filename is unspecified the name will be derived from the uploaded file. If Force is not set to 1 and a file already exists with the specified name in the specified path, a new filename will be created by adding a 1, 2, 3, etc. to the base filename until a unique name is found. You must use "multipart/form-data" encoding when calling this method so that the upload file can be processed.
SESSION_END()
Ends the session by calling Session.Abandon destroying all resources and session information. 
SESSION_START(User, Password)
Initializes a ANGEL user session using the provided username and password. This is not required for using any of the API functions, but could be useful if you wish to directly access user-level pages within the ANGEL environment, especially those within the /Profile directory.
USER_ENROLLMENTS(User)
Returns a list of enrollment entries for the specified user indicating the user's enrollment status in courses, groups and libraries. 
VALIDATE_ACCOUNT(User, Password)
Checks if the specified credentials are valid and returns succes or error result information.
